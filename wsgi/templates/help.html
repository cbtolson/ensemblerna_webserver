File Formats


1) FASTA format
For FASTA, the first line is the title and needs to start with ">". Subsequent lines should only contain sequence (whitespace is ignored). Lowercase nucleotides will be forced single stranded in structure prediction. The fasta file should contain a single sequence. The accepted file extension is .fa.

  >sequence title
  AAAGCUAAGCAUGAAAA

2) SHAPE format
For SHAPE, the file is organized in two columns. The first column is the nucleotide number, and the second is the reactivity. Nucleotides for which there is no SHAPE data can either be left out of the file, or the reactivity can be entered as -999. Columns are separated by any white space. There is no header. The SHAPE file should contain a single trace. The accepted file extension is .shape.

   9  -999
  10  -999
  11  0.042816
  12  0
  13  0.15027
  14  0.16201

3) Dot-Bracket format
For dot-bracket, each line contains a structure in dot-bracket notation. There should be no headers or sequence information and the file should contain an ensemble of structures. The accepted file extension is .db

  ((((...))))...((((...))))
  ((((...((((...))))...))))
  ((((...((((......))))))))



Inputs


Required Inputs
1) FASTA file
RNA sequence of interest for visualization. Fasta file format is required. This file is required for a new visualization. Note: At this time the EnsembleRNA Webserver is limited to sequences shorter than 400 nts. For longer RNAs, please download the EnsembleRNA Python Package.

Optional Inputs
1) Email Address
Email address to receive link to results. If an email address is not supplied, leaving the loading page will result in your visualization not being saved. Recommended option for longer RNAs.

2) SHAPE file
SHAPE trace for the RNA sequence of interest. SHAPE file format is required. If Dot-Bracket file is included, this file will be ignored.

3) Dot-Bracket file
Set of dot-bracket structures for the RNA sequence of interest. Dot-bracket file format is required. This file is optional.

4) map FASTA file
reference RNA sequence for creating a map of conformational space for all other projections. This file should be the same for all sequences that you wish to compare. If a map FASTA file is not included, the RNA sequence FASTA file will be used. This sequence may not actually be used in the final map of conformational space, but ensures a consistent space for projection FASTA file format is required.

5) map Dot-Bracket file
Set of dot-bracket structures for the RNA sequence of interest. Dot-bracket file format is required. This file should be the same for all sequences that you wish to compare.

Advanced Inputs
All advanced options are optional.

1) map Size
Number of sequences to be used in the map of conformational space. (Default is 10).

2) Range
Range of nucleotides to visualize. Predicted structures will include the full length of the input RNA but only the given range will be plotted. (Default is 1 to sequence length).

3) Maximum Distance
The maximum number of bases between the two nucleotides in a pair. (Default is no restriction).

Package Only Inputs
Only available for python package available by download

1) Output directory
Directory for output files. The directory must already exist. (Required input).

2) Temperature
Temperature at which the calculation takes place in Kelvin. (Default is 310.15 K).

3) SHAPE Intercept
The intercept used with SHAPE restraints. Ignored if reference dot-bracket file is included. (Default is -0.6 kcal/mol).

4) SHAPE slope
The slope used with SHAPE restraints. Ignored if reference dot-bracket file is included. (Default is 1.8 kcal/mol).

5) Plot Map
Plot the map. Choices are 'T' or 'F', and single quotes must be included. (Default is 'T').

6) Plot Interactive
Plot the interactive html file. Choices are 'T' or 'F', and single quotes must be included. (Default is 'T').

7) Maximum Threads
Maximum number of threads for multi-threading. (Default is 1).



Output


Interactive Ouput
For more information, see the "Example" page

1) Reference Visualization
Plot for the reference ensemble. Bubble size varies by the number of structures in that cluster. Similar clusters are closer together in space.

2) Mediod Structure
The arc diagram representation of the structure closest to all other structures in the cluster. Veertical gray lines indicate the selected range.

1) Cluster Diversity
Representation of cluster diversity. Clusters with many different unique structures are considered high diversity and clusters with few unique structures are low diversity. The cluster diversity is normalized to the cluster size.

Output Files
For both reference and map of conformational space

1) CSV file
.csv file with cluster number, xy-coordinates, cluster size, mediod structure and vector representation.

2) DB file
.db Dot-Bracket file with structures.

3) PDF file
.pdf file with visualization plot.

4) PNG file
.png file with visualization plot.

5) HTML file
.html file with interactive plotting. (Reference ensemble only).
